## 一、平台概述

### 1.1 产品定位
专为校园师生打造的轻量化失物招领 Web 系统，聚焦失物发布、寻物求助、智能匹配、高效认领四大核心场景。

### 1.2 核心价值
- 内置 AI 智能检索 Agent，解决传统平台"物品多、查找慢、匹配难"的痛点
- 支持 PC/移动端自适应访问
- 无需下载 APP，浏览器直接使用

## 三、功能模块

### 模块1：用户身份认证（校园专属）

#### 1.1 登录/注册
- 支持学号/工号 + 密码注册登录
- 绑定校园身份，杜绝校外人员
- 支持密码找回、手机号绑定（接收认领通知）

#### 1.2 个人中心
| 功能 | 说明 |
|------|------|
| 基础信息 | 姓名（可匿名）、学院、联系方式（隐私保护） |
| 我的发布 | 分类展示「我发布的失物」「我发布的寻物」，支持编辑/删除/标记已找回 |
| 我的认领 | 查看他人对自己物品的认领申请、AI匹配记录 |
| 消息通知 | 「物品被匹配」「有人申请认领」「认领成功」提醒 |

---

### 模块2：失物发布（捡到物品，等待认领）

#### 2.1 发布入口
首页醒目「我捡到了」按钮，一键进入发布表单。

#### 2.2 发布字段
| 字段 | 类型 | 说明 |
|------|------|------|
| 物品分类 | 必填 | 饰品/文具/电子设备/证件/衣物/钱包/其他 |
| 物品名称 | 必填 | 如「黑色发卡」「校园卡」「无线耳机」 |
| 物品照片 | 选填 | 支持上传1-3张实拍图 |
| 捡到时间 | 必填 | 日期+时段（如：10月28日下午） |
| 捡到地点 | 必填 | 教学楼/食堂/图书馆/操场/宿舍区，支持自定义 |
| 物品描述 | 选填 | 详细特征描述 |
| 存放地点 | 必填 | 失物暂存处（如：图书馆前台） |
| 匿名发布 | 可选 | 隐藏个人信息 |

#### 2.3 状态流转
```
待认领（默认） → 已认领（手动标记） → 已失效（超时未认领）
```

---

### 模块3：寻物发布（丢失物品，求助寻找）

#### 3.1 发布入口
首页「我丢失了」按钮，快速发布寻物信息。

#### 3.2 发布字段
| 字段 | 类型 | 说明 |
|------|------|------|
| 物品分类 | 必填 | 同失物分类 |
| 物品名称 | 必填 | 如「粉色星星发卡」 |
| 物品描述 | 必填 | 详细特征（颜色、特征、丢失时间地点） |
| 丢失时间 | 必填 | 日期+时段 |
| 丢失地点 | 必填 | 精确填写 |
| 悬赏意愿 | 选填 | 如必有重谢/无偿感谢 |
| 联系方式 | 必填 | 仅平台可见 |

#### 3.3 状态流转
```
寻找中（默认） → 已找回（手动标记） → 已取消
```

#### 3.4 寻物通知功能（方案B：站内信通知）
当用户看到他人的寻物启事时，如果自己捡到了该物品，可以通知发布者：
- 入口：物品详情页「我捡到了」按钮
- 填写内容：捡到地点、捡到时间、联系方式、补充说明
- 通知类型：`FOUND_NOTIFY`
- 发布者收到站内信，包含详细的捡到信息和联系方式

### 模块4：物品列表展示

#### 4.1 双列表分离
- **失物招领**：他人捡到的物品，等待失主
- **寻物启事**：他人丢失的物品，寻找捡到者

#### 4.2 筛选功能
| 筛选维度 | 选项 |
|----------|------|
| 按分类 | 饰品/电子/证件等 |
| 按时间 | 最新发布/近1天/近3天 |
| 按地点 | 食堂/图书馆/教学楼等 |
| 按状态 | 待认领/已认领/寻找中 |

#### 4.3 展示样式
卡片式布局：缩略图 + 物品名称 + 发布时间 + 地点 + 状态

---

### 模块5：物品详情页

#### 5.1 信息展示
- 照片、分类、时间、地点、详细描述
- 发布人信息（隐私保护）

#### 5.2 操作按钮
| 类型 | 按钮 |
|------|------|
| 失物详情 | 「我要认领」「联系发布人」 |
| 寻物详情 | 「我捡到了」「联系发布人」 |

> 「我捡到了」：当用户看到寻物启事且自己捡到了该物品时，点击后填写捡到信息，系统发送站内信通知发布者

#### 5.3 关联推荐
底部展示 AI 匹配的相似物品（直接跳转）

---

### 模块6：认领流程（安全、防冒领）

#### 6.1 流程步骤
```
申请认领 → 发布人审核 → 确认认领
```

#### 6.2 详细说明
| 步骤 | 操作 | 说明 |
|------|------|------|
| 申请认领 | 用户点击「我要认领」 | 填写物品验证信息（颜色、特征、丢失时间） |
| 发布人审核 | 收到认领申请 | 核对验证信息 + 物品特征 |
| 确认认领 | 核对无误 → 标记「已认领」 | 双方线下交接 |
| 拒绝认领 | 核对失败 → 拒绝申请 | 备注原因 |

#### 6.3 记录留存
- 认领记录
- 聊天记录
- 验证信息
永久留存，防止纠纷。

---

### 模块7：AI智能匹配Agent（核心特色功能）- 最后开发

#### 7.1 AI Agent交互入口
- 平台首页顶部固定AI搜索框
- 提示文字：「描述你丢失的物品，AI帮你精准匹配~」
- 支持文字输入（语音输入可选）

#### 7.2 AI Agent核心功能
| 功能 | 说明 |
|------|------|
| 自然语言理解 | 用户用日常语言描述物品，AI自动提取关键特征 |
| 全库智能检索 | 自动遍历平台所有待认领物品，对比分类、照片、描述、时间、地点 |
| 匹配度排序 | 输出Top5高匹配度物品，按匹配度百分比排序 |
| 匹配理由 | 每条结果标注匹配理由和不匹配原因 |

#### 7.3 实现方案
- 文本匹配：sentence-transformers（免费开源）
- 向量存储：MySQL + 简单向量计算
- 或调用 LLM API（Claude/GPT）辅助分析

### 模块8：后台管理系统（管理员使用）- Phase 7

#### 8.1 用户管理
- 查看/审核校园用户
- 封禁违规账号

#### 8.2 物品管理
- 审核所有发布内容（删除违规/虚假信息）
- 批量处理失效物品

#### 8.3 AI匹配管理
- 查看AI检索日志
- 优化匹配算法

#### 8.4 数据统计
- 每日发布量
- 认领成功率
- 高频丢失物品/地点

#### 8.5 公告管理
- 发布平台通知（如：失物集中认领时间、暂存处地址）

---

### 模块9：通知与提醒

#### 9.1 站内消息
- AI匹配到物品通知
- 认领申请通知（CLAIM_APPLY）
- 审核结果通知（CLAIM_RESULT）
- **物品找到通知（FOUND_NOTIFY）**：当有人捡到发布者丢失的物品时，发送站内信通知，包含捡到地点、时间、联系方式

#### 9.2 短信/微信通知（可选）
- 绑定手机号/微信
- 实时接收提醒

#### 9.3 邮件通知
- 重要认领信息邮件备份

---

### 模块10：通用基础功能

#### 10.1 自适应布局
- PC、手机、平板完美适配
- 校园随手操作

#### 10.2 隐私保护
- 用户联系方式仅认领双方可见
- 不公开个人信息

#### 10.3 反馈功能
- 用户提交bug、建议
- 管理员及时处理

#### 10.4 搜索功能

- 配合AI智能搜索

## 四、数据模型

### 4.1 用户表 ✅ (已实现)
```
users
├── id (主键)
├── username (学号/工号)
├── password (密码)
├── real_name (姓名)
├── college (学院)
├── phone_number (手机号)
├── wx_id (微信)
├── role (USER/ADMIN)
├── is_active (是否启用)
├── created_time
└── updated_time
```

### 4.2 物品表 ✅ (已实现)
```
items
├── id (主键)
├── name (物品名称)
├── description (描述)
├── type (LOST/FOUND)
├── category (分类)
├── location (地点)
├── contact_info (联系方式)
├── found_time (丢失/捡到时间)
├── storage_location (存放地点)
├── images (图片URLs)
├── is_resolved (是否已处理)
├── user_id (发布人ID)
├── created_time
└── updated_time
```

### 4.3 认领表 ✅ (已实现)
```
claims
├── id (主键)
├── item_id (物品ID)
├── user_id (申请人ID)
├── verify_info (验证信息)
├── message (留言)
├── status (PENDING/APPROVED/REJECTED)
├── reject_reason (拒绝原因)
├── created_time
└── updated_time
```

### 4.4 消息表 ✅ (已实现)
```
messages
├── id (主键)
├── user_id (接收用户ID)
├── type (CLAIM_APPLY/CLAIM_RESULT/FOUND_NOTIFY/SYSTEM)
├── content (消息内容)
├── related_id (关联ID)
├── is_read (是否已读)
└── created_time
```

### 4.5 公告表 ✅ (已实现)
```
notices
├── id (主键)
├── title (标题)
├── content (内容)
├── is_active (是否启用)
└── created_time
```
